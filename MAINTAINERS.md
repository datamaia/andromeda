# Maintainers

Maintainers are accountable for subsystem direction, review, and release quality. Review
ownership by path is encoded in [.github/CODEOWNERS](.github/CODEOWNERS); this file is the
human-readable roster.

## Notes

- The project is in its bootstrap phase with a single maintainer. Per RISK-PRD-003 and the
  governance model (Volume 15), reaching **at least two** release-authority holders is a v1
  exit condition; CODEOWNERS entries move from an individual to maintainer **teams** once the
  GitHub organization exists (the organization/namespace is PENDING VALIDATION — Volume 0
  open-questions register, OQ-001).
- To propose yourself as a committer or maintainer, open a Discussion; the criteria and path
  are defined in [Volume 15, chapter 04](docs/spec/volume-15-roadmap-and-execution/04-open-source-governance.md).
