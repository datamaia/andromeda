### ADR-100 — CLI grammar style: noun-group command tree with a closed verb vocabulary

- Status: Accepted
- Date: 2026-07-11
- Deciders: Andromeda specification team
- Components: CLI
- Related requirements: FR-CLI-001, FR-CLI-004, PRD-008, PRD-009

#### Context

The `andromeda` executable exposes more than twenty top-level commands (Volume 8, chapters
03–06) covering execution, configuration, platform management, data inspection, and
maintenance. The brief mandates the top-level command list; it does not mandate how
subcommands, verbs, arguments, and aliases are shaped. Comparable developer tools split into
two grammar families: verb-first trees with idiosyncratic per-command syntax (git's porcelain
grew this way over two decades) and resource-noun trees with a shared verb vocabulary
(kubectl, gh, docker's management commands). Andromeda's surface is authored at once, not
grown, so it can choose a uniform grammar before drift exists. cobra (ADR-005) expresses
either style.

#### Problem

Fix the grammatical shape of the entire command tree — ordering of nouns and verbs, naming
rules, alias policy, depth limits — so that twenty-plus command families authored across
three specification chapters and later extended by third parties (ADR-104) read as one
language rather than twenty dialects.

#### Forces and constraints

- Predictability is measurable: a user who knows `provider list` should guess `plugin list`,
  `skill list`, and `mcp list` correctly on the first attempt.
- The execution family (`run`, `plan`, `exec`, `init`) is action-shaped, not resource-shaped;
  forcing nouns onto it (`andromeda run create`?) would be grammar for grammar's sake.
- Shell completion and help generation (ADR-005) reward regular trees; irregular grammars
  produce irregular completion.
- Command paths are public contract under SM-20: renames are breaking changes, so the naming
  rules must be right before MVP, not refactored after.
- Extension commands (ADR-104) must be nameable without colliding with future built-in nouns.

#### Alternatives considered

1. **Resource-noun groups with a closed shared verb vocabulary, plus action leaf commands
   (selected).** Uniform where the surface is resource management; direct where it is
   execution.
2. **Verb-first grammar** (`andromeda list providers`, `andromeda install plugin X`) —
   rejected: verbs take heterogeneous object grammars, completion becomes object-dependent
   at position 2, and the tree cannot delegate a whole resource family to one owner chapter
   cleanly.
3. **Free per-command grammar** (each command family designs its own) — rejected: this is
   how inconsistency ships; the git porcelain's `git branch -d` / `git remote remove`
   divergence is the canonical cautionary example.
4. **Everything resource-shaped including execution** (`andromeda run create --goal …`) —
   rejected: it buries the product's primary action behind ceremony and reads worse in every
   example the specification produces.

#### Decision

1. The command tree uses **resource-noun groups** (`provider`, `model`, `tool`, `plugin`,
   `skill`, `workflow`, `mcp`, `memory`, `context`, `index`, `session`, `config`, `auth`,
   `git`, `trace`) whose subcommands come from a **closed shared verb vocabulary**, and
   **action leaf commands** (`run`, `plan`, `exec`, `init`, `logs`, `export`, `doctor`,
   `update`, `version`, `completion`) for direct operations.
2. Shared verbs, one meaning each, corpus-wide: `list`, `show`, `add`, `remove`, `install`,
   `uninstall`, `enable`, `disable`, `test`, `status`, `validate`, `search`, `export`.
   Domain verbs beyond this set (e.g., `commit`, `rotate`, `pin`) are permitted only inside
   the group whose domain owns the term, and are recorded in FR-CLI-001. Extending the shared
   vocabulary requires amending FR-CLI-001 through the Volume 0 change procedure.
3. Nouns are singular (`provider`, not `providers`). Command path depth is at most three
   levels below the binary name. All names are lowercase ASCII; multi-word flags are
   kebab-case; no camelCase, no abbreviations of command words.
4. **No aliases at MVP.** Alias introduction is reserved for renames, where the old path
   becomes a DEPRECATED alias for at least one minor release before removal (SM-20 regime).
5. The top-level namespace is closed to extensions: third-party commands mount only under
   the reserved `x` group (ADR-104). `help`, `completion`, and the hidden completion
   machinery are reserved words.

#### Rationale

A closed verb vocabulary converts grammar consistency from a review aspiration into a
checkable rule: the linter and grammar golden tests (Volume 13) can verify that every
resource group's verbs come from the published set. Splitting resource groups from action
leaves respects how users think — "manage my providers" versus "run this goal" — instead of
flattening both into one artificial shape. Closing the top-level namespace now is what makes
ADR-104's extension mounting safe later.

#### Positive consequences

- First-guess correctness across families; one help pattern; one completion pattern.
- The grammar is testable as data: the tree and verb table in FR-CLI-001 are the oracle for
  golden tests.
- Renames are governed events with deprecation windows, not silent breaks.

#### Negative consequences

- Domain verbs must be argued into existence per group, which adds specification friction.
- Some invocations are one word longer than a bespoke grammar would allow.
- The no-alias rule forgoes conveniences users of other tools expect (e.g., `ls` for
  `list`); muscle-memory portability is deliberately sacrificed for a smaller contract
  surface.

#### Risks

- Verb-vocabulary erosion through "just one exception" additions; mitigation: the vocabulary
  lives in FR-CLI-001 under change control, and grammar golden tests fail on undeclared
  verbs.
- Depth-3 limit could pinch a future family (e.g., nested resource hierarchies); mitigation:
  the limit is amendable by ADR, and no mandated command needs depth 4 today.

#### Reversal plan

Grammar style is declarative cobra tree data plus FR-CLI-001. Restyling (e.g., adopting
verb-first) would be a major-version break under SM-20 executed as a table-driven re-mapping
with DEPRECATED aliases for every moved path during the transition window. Blast radius:
Volume 8 chapters 03–06, completion scripts, and documentation — no Runtime change.

#### Review conditions

- Grammar golden tests reveal recurring user error patterns (mis-guessed verbs) in usability
  passes at Beta.
- The shared verb vocabulary accumulates more than five domain-verb exceptions in one
  release cycle — re-examine whether the vocabulary is misfactored.
- ADR-104's `x` namespace proves insufficient for a first-party plugin bundle that warrants
  top-level presence.
