### ADR-001 — Implementation Language: Go

- Status: Accepted
- Date: 2026-07-11
- Deciders: Andromeda specification team
- Components: Core Domain, Runtime, CLI, TUI, Extension SDK, Platform Abstraction Layer (PAL)
- Related requirements: Volume 1 product objectives; formal requirement links are added at consolidation

#### Context

Andromeda is an open-source, local-first, vendor-agnostic AI agent engineering platform delivered as one executable (`andromeda`) containing both the CLI and the TUI, configured via `andromeda.toml`, targeting macOS and Linux first with Windows planned later. Development happens openly on GitHub, with contributions from human engineers and from AI implementation agents. The implementation language shapes every subsequent technical decision: binary distribution, concurrency model, library availability for TUI/MCP/OpenTelemetry, testing strategy, release tooling, and the barrier to entry for contributors.

The project sponsor provided Go as the preferred implementation language. Under the statement classification of Volume 0, this preference is a **provided constraint**. The technology-selection policy nevertheless requires that the language not be chosen arbitrarily: Rust, Go, TypeScript, and Python are evaluated against explicit criteria, and the provided preference is recorded below as one force among others — not as the rationale.

#### Problem

Select a single primary implementation language for the Andromeda codebase — Core Domain, Runtime, CLI, TUI, Extension SDK, and supporting components — that satisfies single-binary distribution, the portability roadmap, the extension and protocol strategy, and open-source contribution goals, and document the comparative evaluation honestly.

#### Forces and constraints

- Distribution MUST be a self-contained binary per platform/architecture with no interpreter or runtime dependency (Volume 14).
- Target platforms: macOS and Linux (amd64/arm64) now; Windows later without re-architecture.
- The workload is I/O-bound by design: streaming HTTP to model providers, subprocess orchestration (plugins per ADR-009, MCP stdio servers per ADR-010), file indexing, and SQLite persistence (ADR-007). Compute-heavy inference runs out of process in providers.
- The TUI is a long-running, event-driven interactive program requiring a mature terminal ecosystem (ADR-006).
- Open development on GitHub favors a language with a low contribution barrier, uniform formatting, and short build/test cycles — this also benefits AI implementation agents, which iterate through compile/test loops.
- Provided constraint: sponsor preference for Go.
- Web-verified ecosystem facts (2026-07-11): official MCP Go SDK stable v1.6.1 under the `modelcontextprotocol` org; Charm TUI stack v2 active; `modernc.org/sqlite` v1.53.0 CGO-free; official OpenAI Go SDK v3.41.1 and Anthropic Go SDK v1.57.0 both GA; goreleaser v2.17.0 active.

- **Rust.** Strongest on memory safety (compile-time freedom from data races and use-after-free) and raw performance; excellent single-binary story; `ratatui` is a capable TUI ecosystem. Costs: the steepest learning curve of the four, the slowest compile times, a materially higher contribution barrier for an open project, and — at decision time — no first-party Anthropic/OpenAI/MCP SDK maturity was verified for Rust, so those rows are scored conservatively rather than asserted.
- **TypeScript.** Official MCP tooling and provider SDKs exist, and the async model fits streaming. Disqualifying weakness: distribution. Shipping a CLI/TUI requires bundling a JavaScript runtime or using single-binary packagers, inflating artifact size and complicating notarization, updates, and the future Windows matrix. Runtime type erasure also weakens contract enforcement at the protocol boundaries Andromeda depends on.
- **Python.** Best-in-class ML/embeddings ecosystem and gentle onboarding. Disqualifying weakness: packaging and performance. Interpreter dependency, virtual environments, and native-extension wheels contradict the single-binary requirement; the GIL constrains the concurrency model the Runtime needs for parallel tool execution and streaming.
- **Go (selected).** See Decision and Rationale.

#### Decision

Andromeda MUST be implemented in Go. The minimum toolchain at adoption is Go 1.24, driven by the verified requirement of `github.com/anthropics/anthropic-sdk-go` (Go ≥ 1.24). Go is the single primary language for all product components; auxiliary tooling (CI scripts, packaging glue) MAY use shell or Python. Introducing another language into the shipped runtime requires a new ADR. Extensions are explicitly polyglot: plugins communicate over the language-agnostic Andromeda Runtime Protocol (ADR-009), so the Go choice does not constrain extension authors.

#### Rationale

1. **The workload profile neutralizes Rust's main advantage.** Andromeda's hot paths are dominated by network latency to providers, token streaming, and subprocess I/O. Go's performance (scored 4) is not the bottleneck in any identified scenario; Volume 12 sets measurable targets that gate this claim (see Review conditions).
2. **Concurrency model fits the architecture.** Goroutines and channels map directly onto the Event Bus (ADR-012), the Task Scheduler, streaming provider responses, and supervising plugin/MCP subprocesses, without an async-runtime dependency or function-coloring split.
3. **Verified, subsystem-complete library coverage.** Every foundational subsystem has a verified, actively maintained Go library: Charm v2 TUI stack (ADR-006), `modernc.org/sqlite` CGO-free driver (ADR-007), `pelletier/go-toml/v2` (ADR-008), the official and stable MCP Go SDK v1.6.1 (ADR-010), OpenTelemetry Go SDK and stdlib `log/slog` (ADR-011), and goreleaser/cosign/syft (ADR-013). No other candidate language had this full set verified.
4. **Distribution and portability.** CGO-free static binaries cross-compile from a single CI host to the whole macOS/Linux matrix, and later Windows, with `GOOS`/`GOARCH` alone — the property that makes ADR-013's release pipeline simple to specify.
5. **Contribution accessibility and maintainability.** One canonical formatter, a small language surface, and second-scale builds shorten the loop for human and AI contributors alike, and keep long-term maintenance tractable.
6. **The provided constraint and the evaluation converge.** The sponsor preference is a documented force; the scored evaluation reaches the same outcome independently, with Go leading Rust 97 to 83 on the briefed, unweighted criteria.

Honest concession: Rust is superior on memory safety and peak performance. Go retains bounds checking and garbage collection (eliminating C-class memory corruption); the residual gap is runtime data races, mitigated by mandatory `-race` execution in CI (ADR-017) and channel-oriented ownership design.

#### Positive consequences

- Single static binary per platform; trivially scriptable cross-compilation and release automation.
- One language across Core Domain, Runtime, CLI, and TUI; no polyglot build system in the product tree.
- Direct, verified SDK support for MCP, both MVP cloud providers, and OpenTelemetry.
- Short compile/test cycles improve throughput for AI-agent-driven contributions.

#### Negative consequences

- Garbage-collector pauses exist, though they are not expected to be user-visible in an I/O-bound TUI workload; Volume 12 defines the latency budgets that verify this.
- No compile-time data-race prevention; discipline and race-detector CI are required.
- Weaker native ML/embeddings ecosystem than Python; Andromeda compensates by sourcing embeddings from providers over HTTP (Ollama `/api/embed`), per the MVP provider scope.
- Error-handling verbosity and less expressive generics than Rust or TypeScript.

#### Risks

- A critical Go dependency (Charm stack, MCP SDK) could stall; mitigated by pinning, by port interfaces around each library, and by the review conditions below.
- GC or scheduler behavior could breach Volume 12 latency targets under extreme indexing loads; mitigated by benchmark gates before each phase exit.

#### Reversal plan

Full reversal is a rewrite and is treated as a last resort. The architecture deliberately keeps reversal tractable at the subsystem level: the Andromeda Runtime Protocol (ADR-009), MCP (ADR-010), the SQLite schema (ADR-007), and `andromeda.toml` (ADR-008) are all language-neutral contracts, so any single component can be re-implemented in another language as a subprocess behind the same protocol without breaking the rest. If a full migration ever becomes necessary, it proceeds component-by-component along these seams, starting with the component that triggered the reversal.

#### Review conditions

- Volume 12 benchmarks attribute a missed performance or memory target to the Go runtime itself (not to a replaceable library) in two consecutive releases.
- The official MCP Go SDK or the Charm v2 stack goes unmaintained (no release for more than 12 months) with no viable fork.
- Entry into the Windows phase invalidates the CGO-free portability assumptions recorded here.
- The unweighted scoring assumptions change materially (e.g., a first-party Rust SDK set reaches verified GA parity) at the time of a major-version re-architecture.
