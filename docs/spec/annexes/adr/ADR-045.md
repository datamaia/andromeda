### ADR-045 — Sub-agent delegation: bounded depth, permission narrowing, budget slices

- Status: Accepted
- Date: 2026-07-11
- Deciders: Andromeda specification team
- Components: Agent Engine, Permission Manager, Execution Engine, Task Scheduler
- Related requirements: FR-AGT-004, FR-AGT-005; PRD-005, PRD-006

#### Context

Volume 2 models Agents as run-bound instances with an acyclic delegation tree
(`parent_agent_id`, INV-AG-01/02). Multi-agent patterns (explorer/executor/reviewer splits)
reduce context pressure and enable parallel tasks, but every added instance is added
autonomous authority. MVP runs are single-agent (Volume 1 item 4); delegation lands at
Beta.

#### Problem

Under what containment rules may an agent create and direct other agents?

#### Forces and constraints

- Least privilege: delegation must never widen what the run could do (PRD-005).
- Accountability: every sub-agent action must chain to its delegating agent and run
  (PRD-006, SM-13).
- Runaway prevention: recursive delegation is a fork-bomb shape (RISK-AGT-001).
- Budgets are run-level facts (Volume 2); delegation must not create unaccounted spend.

#### Alternatives considered

1. **No delegation ever** — rejected: forfeits real decomposition value; the entity model
   already supports instances cleanly.
2. **Free delegation with shared run authority** — rejected: authority ambient to the run
   would let a compromised sub-agent use the parent's full grant set; audit trees become
   forests.
3. **Sub-agents as separate runs** — rejected: fractures the causal record of one goal
   across runs, breaking SM-12/SM-13 chains and INV-RUN invariants for no containment gain.
4. **Same-run instances with depth bound, server-side permission narrowing, and explicit
   budget slices** — chosen.

#### Decision

Delegation creates Agent instances in the same run, tree-linked per INV-AG-01/02, with:
depth bounded by `agent.loop.max_subagent_depth`; the child's permission context
constructed by the Permission Manager as parent-context-minus-narrowing (never
cooperative); an explicit budget slice deducted from run budgets; results returned to the
parent as structured messages; the child's lifecycle per the Agent machine
(chapter 05). Delegation is disabled by default until Beta (FR-AGT-004).

#### Rationale

Same-run instances preserve the single causal record; server-side narrowing makes least
privilege an enforced property rather than a convention; slices keep FR-AGT-005 accounting
exact; the depth bound converts recursion into a data-refused request.

#### Positive consequences

- One audit chain per goal, delegation included (SM-13 holds).
- Compromise blast radius shrinks with each delegation level.
- Parallel task execution (chapter 03) gets its agent supply without new entities.

#### Negative consequences

- No cross-run or cross-session agent collaboration (out of this decision's scope).
- Slice sizing adds a planning burden for parents.

#### Risks

- Narrowing bugs would be privilege escalation — covered by Volume 9 enforcement tests and
  the FR-AGT-004 widening fixtures.

#### Reversal plan

Delegation is additive behavior over existing entities; disabling it (configuration) or
removing it reverts to single-agent runs without schema changes.

#### Review conditions

Review at Beta exit with field data on depth utilization and slice friction; review if
workflow-level multi-agent orchestration (FR-WF-001) subsumes common delegation patterns.
