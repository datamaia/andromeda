### ADR-105 — Danger token fixed at `#FF6B6B` with a contrast-validated light-theme variant

- Status: Accepted
- Date: 2026-07-11
- Deciders: Andromeda specification team
- Components: TUI, CLI
- Related requirements: FR-UX-040, FR-UX-042, NFR-UX-040; completes the ADR-026 PENDING VALIDATION on the Danger value

#### Context

ADR-026 fixed four brand color tokens (Primary `#7C5CFF`, Secondary `#8C7B6E`, Tertiary
`#F5F2ED`, Neutral `#121417`) and deliberately left Danger open: "an accessible soft red
whose exact value Volume 8 fixes — PENDING VALIDATION against a WCAG-style contrast
criterion on the token backgrounds." Volume 8 chapter 08 owns the token tables and the
tier mappings; the Danger value is needed by error rendering, destructive-action styling,
permission prompts, and diff removals across both the TUI and the CLI.

#### Problem

Choose the exact Danger value(s) such that danger-classified text meets a 4.5:1 WCAG 2.x
contrast ratio on both theme backgrounds (Neutral `#121417` dark; Tertiary `#F5F2ED`
light), reads as a "soft" red consistent with the brand board's warm, muted register, and
quantizes acceptably to the 256-color and 16-color tiers.

#### Forces and constraints

- The 4.5:1 text criterion binds on both backgrounds; a single hue value cannot satisfy
  both (a red bright enough for near-black backgrounds is necessarily too light for
  off-white backgrounds).
- "Soft" red: the brand register is warm and low-saturation-leaning; pure `#FF0000`-class
  reds clash with the violet/taupe palette and read as alarm rather than information.
- The value must survive quantization: its nearest 256-color cube entry must retain the
  contrast property against the tier's background entry.
- Danger must never be color-only (color vision deficiencies, monochrome tiers) — a
  companion rule, not a color choice, but it bounds how much work the hue must do.

#### Alternatives considered

1. **`#FF6B6B` dark / `#B3261E` light (selected).** Computed ratios: 6.65:1 on Neutral;
   5.85:1 on Tertiary; 6.29:1 for the 256-tier mapping (index 203 on 233); 6.41:1 for the
   light 256-tier mapping (index 124 on 255). Soft coral-leaning red, harmonious with the
   warm palette.
2. **Single value `#E85D5D` for both themes** — 5.41:1 on Neutral but only 3.1-class on
   Tertiary; fails the light-theme text criterion. Rejected: one value cannot pass both
   backgrounds without being neither soft nor red.
3. **Brighter `#FF7A70`** — 7.27:1 on Neutral, more headroom, but visibly orange-shifted
   and lighter than the brand register; its light-theme problem is identical to 1 (still
   needs a dark variant). Rejected on brand-fit grounds; no accessibility advantage that
   matters past 4.5:1 with margin.
4. **Material-style `#B3261E` for both themes** — passes light (5.85:1) but only 2.5:1 on
   Neutral; unusable on dark. Rejected; retained as the light-theme variant where it
   excels.

#### Decision

The **Danger token is `#FF6B6B`**. Themes resolve it as follows: the dark theme renders
Danger as `#FF6B6B` (6.65:1 on Neutral `#121417`); the light theme renders Danger through
the derived value **`#B3261E`** (5.85:1 on Tertiary `#F5F2ED`), exactly as Primary and
Secondary resolve to per-theme readable variants in chapter 08's derived palette. Tier
mappings: 256-color index 203 (dark) / 124 (light); 16-color ANSI 9 (dark) / 1 (light).
Danger-classified elements always carry a textual marker in addition to color
(NFR-UX-040). This closes ADR-026's PENDING VALIDATION item on the Danger value.

#### Rationale

The pair passes the 4.5:1 criterion with meaningful margin on both backgrounds — margin
matters because real terminals approximate colors — and survives 256-color quantization
with the property intact (6.29:1 and 6.41:1 against the tier backgrounds). `#FF6B6B` is
the softest of the candidates that keeps ≥ 6:1 on Neutral, matching the "accessible soft
red" brief. Making the light value a theme-resolved derivative rather than a second token
keeps the token vocabulary at five names, consistent with ADR-026's closed-set principle
and with how the other chromatic tokens already behave per theme.

#### Positive consequences

- Every danger rendering in the corpus now has one exact, testable expected value per
  theme and tier; the contrast scanner can gate regressions mechanically.
- ADR-026's open item closes; no surface needs to hedge on danger styling.
- The value harmonizes with Primary violet on the same dark background (both sit in the
  6–7:1 band, so mixed error/accent lines render with even visual weight).

#### Negative consequences

- Two concrete values to maintain (dark and light renderings of one token); a rebrand of
  Danger touches both plus four tier table entries.
- The light variant `#B3261E` is visibly deeper than the dark variant; brand collateral
  that shows both side by side must explain the theme-resolution rule.

#### Risks

- At the 16-color tier, ANSI 9/1 are user-palette-defined and can fall below any ratio;
  mitigated by the never-color-only rule and RISK-UX-040's documented variance handling.
- Future brand-board revisions could re-tint the palette and strand the chosen red;
  mitigated by the closed-vocabulary change procedure — a re-tint is an explicit
  superseding decision, not drift.

#### Reversal plan

Danger is referenced by role name everywhere (FR-UX-040 prohibits literals), so replacing
the value is a two-cell change in chapter 08's tables plus tier rows and regenerated
golden frames; no consuming code changes. A superseding ADR records any replacement.

#### Review conditions

- Brand assets are redesigned (ADR-026's own review condition triggers this ADR too).
- Accessibility guidance adopted by the project moves beyond WCAG 2.x contrast arithmetic
  (e.g., a perceptual model) — re-validate the pair under the new criterion.
- Terminal-matrix testing (chapter 12) reveals systematic rendering of the chosen values
  below criterion on popular emulators.
